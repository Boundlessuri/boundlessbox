# 2026 世界杯赛程预测工具 — 设计文档

**日期:** 2026-06-14  
**状态:** 设计中

## 概述

一个 PyQt6 桌面应用，展示 2026 美加墨世界杯赛程，基于过去 12 年 H2H 交手记录和 Elo 评分预测比赛比分。支持混合/纯 H2H/纯 Elo 三种预测模式，打包为独立 exe。

## 数据层

### 数据来源与规模

| 赛事类型 | 覆盖年份 | 预估场次 | 权重 | 来源 |
|----------|----------|----------|------|------|
| 世界杯正赛 | 2014/2018/2022 | ~192 | ×1.0 | FBref/内置 |
| 洲际杯 (欧洲杯/美洲杯/亚洲杯/非洲杯等) | 2015-2025 | ~800 | ×0.8 | FBref/内置 |
| 友谊赛 (FIFA认可) | 2014-2026 | ~1500 | ×0.5 | FIFA.com/内置 |
| U20 世界杯 / U23 奥运 | 2015-2024 | ~400 | ×0.4 | FIFA.com/内置 |
| U17 世界杯 | 2015-2023 | ~300 | ×0.35 | FIFA.com/内置 |
| **合计** | | **~3200+ 场** | | |

### 时间衰减

```
weight = base_weight × 0.85^(years_ago)
```

- 2026 年比赛: ×1.0
- 2022 年比赛: ×0.85^4 ≈ 0.52
- 2018 年比赛: ×0.85^8 ≈ 0.27
- 2014 年比赛: ×0.85^12 ≈ 0.14

### 梯队加成

U23/U20/U17 比赛中，若球员当时 ≤23 岁且现在 ≤27 岁（即现在国家队适龄），该场比赛权重额外 ×1.2。4 年后的 U23 主力大概率是现在的主力。

### 数据准确性保证

1. **内置数据** — 从 FBref.com（足球统计权威来源）批量抓取，结果校验后打包
2. **淘汰赛赛程** — 从 FIFA.com 或 ESPN 实时抓取，含 fallback 到内置预测赛程
3. **数据更新** — 菜单栏 "更新数据" 按钮一键重抓
4. **校验** — 每场比赛记录来源 URL，可追溯

## 预测引擎

### H2H 模型

对每场比赛 (主队 H vs 客队 A)：

```
H进球期望 = H近N场对A类对手场均进球 × H2H加成 × 主场系数
A进球期望 = A近N场对H类对手场均进球 × H2H加成 × 客场系数

主/客场系数: 主 1.15, 客 0.85, 中立 1.0
```

H2H 直接交手记录的贡献：
```
H2H加成 = 1 + (H对A的H2H胜率 - 0.5) × 0.3
```

### Elo + 泊松模型

1. 每队维护 Elo 分（初始基于 FIFA 排名）
2. 比赛后按结果 + 净胜球更新 Elo
3. 用主客队 Elo 差换算预期进球 λ_h, λ_a
4. 泊松分布 P(λ, k) 计算各比分概率
5. 取期望值最高的比分

```
Elo更新: Δ = K × (实际 - 预期), K=40 (大赛)
λ_h = league_avg × exp((Elo_h - Elo_a - γ) / 400)
λ_a = league_avg × exp((Elo_a - Elo_h + γ) / 400)
γ = 主场优势 100 Elo 分
```

### 融合模式

```
最终进球 = H2H预测 × w + Elo预测 × (1-w)
默认 w = 0.5，用户可在设置中拖动滑块调整
```

### 输出

对每场比赛输出：H2H 预测比分、Elo 预测比分、融合比分、各比分概率 TOP3。

### 数据文件嵌入

使用 PyInstaller `--add-data` 将 JSON 数据文件嵌入 exe，运行时通过 `sys._MEIPASS` 读取。

### 数据采集策略

1. **小组赛赛程** — 手工整理 12 组 × 4 队 + 72 场小组赛，直接写入 `group_stage.json`
2. **H2H 历史库** — 编写采集脚本 `data/collector.py`，从 FBref.com 批量抓取，输出 `h2h_database.json`
   - 按球队逐一抓取 2014-2026 所有国际比赛
   - 对 48 支参赛队 + TOP20 常碰面队，共约 70 队
   - 采集后去重、校验（主客进球总和不为负、日期无未来）
3. **Elo 初始分** — 基于 2026 年 4 月 FIFA 排名换算
4. **淘汰赛赛程** — 运行时爬取，若网络失败则用预估对阵（小组第一 vs 小组第二模板）

### 错误处理

| 场景 | 处理 |
|------|------|
| 网络断开 | 淘汰赛区域显示"网络不可用，显示预估对阵"，状态栏变黄 |
| 爬虫解析失败 | 保留上次成功数据，弹窗提示"数据更新失败，使用缓存" |
| H2H 无交手记录 | 显示"两队近12年无交手"，仅用 Elo 预测 |
| JSON 数据文件损坏 | 启动时校验 schema，损坏则弹窗退出 + 提示重装 |
| 打包后路径错误 | 统一用 `sys._MEIPASS` fallback 到 `os.path.dirname(__file__)` |

### 测试策略

- **单元测试**: H2H 加权计算、Elo 更新公式、泊松概率分布
- **集成测试**: 爬虫 → 解析 → 入库 → 预测 全链路
- **GUI 冒烟测试**: 启动 → 点击比赛 → 切换模式 → 打开设置 → 保存
